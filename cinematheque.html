How It Works

Technical Notes

Architecture

Cinematheque uses a content-based filtering model running entirely in the browser. Each film is represented as a feature vector drawn from its genres, director, cast, decade, and keywords. Similarity is computed via a weighted Jaccard / cosine hybrid and ranked to surface the closest matches.

Embedded Dataset → Feature Extraction → TF-IDF Weighting → Cosine Similarity → Top-K Results

Feature Engineering

Each film's "soup" string combines: genres (weighted ×3), director (weighted ×2), cast names, decade tag, and thematic keywords. The TF-IDF vectoriser down-weights ubiquitous terms like "drama" and "thriller" that appear in almost every film, so rarer shared features drive similarity scores higher.

Fuzzy Title Matching

The search input uses character-level edit distance (Levenshtein) to handle typos and partial titles, so "godfather" matches "The Godfather" and "bladerunner" matches "Blade Runner".
